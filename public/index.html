📚 Schema Stacking Strategy & Guidance

          
Advanced schema implementation strategy for maximum Knowledge Graph impact

        

        

          
          

            
🏆 The Four Most Powerful Schema Properties

            
According to advanced schema research, these four properties have the most impact on your Knowledge Graph presence:

            

              

                1. sameAs
                
Link to ALL citations, directories, social profiles, Wikipedia, Wikidata, and press mentions. The more authoritative links, the stronger your entity footprint.

              

              

                2. knowsAbout
                
Link to Wikipedia/Wikidata pages for every topic your business is an expert in. Use Wikidata Q-codes (Google Machine IDs) for maximum entity strength.

              

              

                3. areaServed
                
Define your service areas with rich geo-circle data. Tie locations to your brand with descriptive text that helps Google understand your geographic reach.

              

              

                4. mentions
                
Link to competitors, partners, tools, and industry concepts. This builds Knowledge Graph connections and establishes your entity in the broader ecosystem.

              

            

          


          
          

            
🏗️ The 3-Layer Schema Stack

            
Layer your schema across different page types for maximum Knowledge Graph coverage:

            

              

                
📍 Layer 1: Sitewide Schema

                
    
                  
  • Organization - Core business entity with sameAs, knowsAbout, mentions
    • 
                  
  • WebSite - With SearchAction for sitelinks searchbox
    • 
                

                
✅ Include on EVERY page of your site

              

              

                
📄 Layer 2: Service Page Schema

                
    
                  
  • Service - With areaServed, provider, offers
    • 
                  
  • Product - For product/service pages
    • 
                  
  • FAQPage - Only if visible FAQs exist!
    • 
                

                
✅ Add to specific service/product pages

              

              

                
🏠 Layer 3: Local Page Schema

                
    
                  
  • LocalBusiness - With geo, openingHours, areaServed
    • 
                  
  • HowTo - For tutorial/guide pages
    • 
                  
  • Article/BlogPosting - For blog content
    • 
                

                
✅ Add to location-specific and content pages

              

            

          


          
          

            
🔗 Entity Linking Best Practices

            

              

                
Google Machine IDs (Wikidata Q-codes)

                
Use Wikidata Q-codes for the most powerful entity linking. Find Q-codes at wikidata.org by searching for your topic. Example: SEO = Q316680

              

              

                
Wikipedia URLs

                
Link to Wikipedia articles for topics, people, and organizations. This connects your schema to established Knowledge Graph entities.

              

              

                
@id References

                
Use @id references to link entities instead of duplicating. Your Organization should have one canonical @id used across all pages.

              

            

          


          
          

            
⚠️ Critical Warnings

            

              

                FAQPage Requires Visible Content
                
Only add FAQPage schema if there are actual Q&As visible on the page. Google penalizes mismatched FAQ schema!

              

              

                Homepage vs Sitewide Schema
                
Organization schema on homepage should be your canonical entity. Service pages should reference it via @id, not duplicate it.

              

              

                Don't Invent Data
                
Never invent ratings, prices, dates, or author information. Only include data that actually exists and can be verified.

              

              

                HowTo Requires Step Structure
                
Only add HowTo schema if your page has actual step-by-step instructions. The steps must be visible in the page content.
